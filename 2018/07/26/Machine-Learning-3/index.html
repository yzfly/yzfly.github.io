机器学习之泛化

泛化能力： 是指机器学习模型对新数据的适应能力。

监督式机器学习基于几条重要假设，若违背了这几条重要假设，模型将不具备普适性-也就是不具备泛化能力，不能很好的运用到训练数据集之外的数据。

机器学习三条基本假设

  • 从分布中随机抽取独立同分布（i,i,d)的样本。换言之，样本之间不会相互影响。
  • 分布是平稳的；即分布在数据集中不会发生变化。
  • 从同一分布的数据划分中抽取样本。

如果不符合监督式机器学习的关键假设，那么模型将失去具备泛化能力的理论保证。

如果某个模型尝试紧密拟合训练数据，却不能够很好的泛化到新数据，就会发生过拟合现象。

过拟合

机器学习的目标是对从真实概率分布（已隐藏）中抽取的新数据做出良好预测。模型需要适当拟合当前数据，但也要尽可能简单的拟合数据。

模型的如果过于复杂，即使在训练过程中产生的损失很低，但在预测新数据方面可能表现糟糕。

为了评估模型的拟合情况，是否发生过拟合现象，采用奥卡姆剃刀定律：

机器学习模型越简单，良好的实证结果就越有可能不仅仅基于样本的特性。

考虑以下两点因素：

  • 模型的复杂程度
  • 模型在处理训练数据方面的表现

机器学习模型旨在根据以前未见过的新数据做出良好预测。上面的讨论给处理理论评估方法，而机器学习侧重于实证评估来评判模型泛化到新数据的能力。采用分割数据集为两个子集的方法来评估：

  • 训练集-用于训练模型的子集，越大模型越准确
    *
  • 测试集-用于测试模型的子集，越大置信区间越好
    • 规模足够大，可产生具有统计意义的结果
    • 能代表整个数据集。换言之，测试集特征和训练集的特征相同。

绝对不要对测试集进行训练： 如果在测试集上表现过于良好，因检查是否存在此问题，尤其是数据集中存在大量冗余数据的情况下，很容易犯错。因此预处理数据时应该先去除冗余数据。

采用模型在测试集上的表现来衡量模型的泛化能力。

验证

一般的方法将数据划分为两个子集，一个子集用于训练，一个子集用于测试。根据模型在测试集中的表现来调整模型的各个参数，最终得到在测试集上表现较好的一个结果。但是重复调整的次数越多，模型的泛化能力就越低，因为测试集暴露给模型的信息也越多。

可引入验证集，使测试集暴露给模型的信息更少，来减少过拟合的风险，提高模型的泛化能力。

算法流程：

  • 划分数据集为三个子集：训练集、验证集、测试集
  • 在训练集上训练，使用验证集进行验证，直到模型在验证集上表现良好
  • 将上一步训练好的模型用于测试集，评估模型的泛化能力。
